## 3. Как HTML должен загружать данные

Внутри `<name>.html` обязательно используется относительный fetch,
поднимающийся на уровень выше в `data/`, для КАЖДОГО типа данных
(items, пресеты весов, footer-примечания) — все три загружаются
параллельно через Promise.all:

```javascript
Promise.all([
  loadCSV('../data/<name>_items.csv'),
  loadCSV('../data/<name>_age_weight_presets.csv'),
  fetch('../data/<name>_footer_notes.txt').then(r=>{
    if(!r.ok) throw new Error('HTTP '+r.status);
    return r.text();
  })

     Проверь, что файлы лежат в data/ и сайт открыт через GitHub Pages,
     а не как локальный файл.</div>`;
});
```

Важно: fetch с относительным путём работает только через HTTP(S)
(например GitHub Pages), а не при открытии файла напрямую с диска
(`file://`) — при локальном открытии браузер блокирует fetch на CSV/TXT
из соображений безопасности (CORS). Для локальной проверки нужен
локальный веб-сервер (например `python -m http.server`).

Правило "три файла — три fetch": ни данные моделей, ни веса пресетов,
ни текст подвала НЕ хранятся внутри HTML в виде строк или JSON —
только в отдельных файлах data/. Это касается любых будущих текстовых
блоков (инструкции, легенды и т.п.) — всё выносится в отдельный .txt.
